# 롤 자기관리 트래커 (LoL Tracker)

연패 추격·판수 폭주·심야 롤 같은 **도박성 패턴**을, 내가 키우는 데스크톱 캐릭터(**몽뿔이**)가 감지해서 말려주는 자기구속(commitment device) 프로그램.

핵심은 알림이 아니라 **캐릭터**다. "내가 키운 정든 캐릭터가 나 때문에 화내고, 게임 시작 버튼 앞을 몸으로 막아서는" 감정적 무게(다마고치 죄책감)가 행동 변화의 동력이다. 설계 배경은 `lol_tracker_handoff.md` 참고.

## 3층 구조

```
[3층] 캐릭터        ← 화면에 상주, 화내고 막아서고, 여의주 모아 진화
   ↑
[2층] 실시간 감지    ← LCU API. 로비/매칭/인게임 상태를 실시간으로 잡음
   ↑
[1층] 통계 (기반)    ← Riot 웹 API. 판수/연패/역전패 계산. 모든 판단의 재료
```

## 주요 기능

**감지·개입**
- LCU `gameflow-phase` 실시간 구독 → 로비 진입/큐 돌리기/매칭 잡힘/챔픽/인게임 상태를 잡음
- Riot 웹 API(Match-V5)로 오늘 판수·현재 연패 계산 (리메이크 제외, 새벽 리셋)
- 한도(판수·연패) 초과 시 캐릭터가 **게임 찾기 버튼 위로 이동해 몸으로 막음** — 창을 넓혀 클릭을 삼키고, 드래그 잠금, 클라를 옮겨도 실시간 추적해 따라붙음
- 통과 조건은 둘뿐: **① 명상 완주 ② 할 일 전부 끝내기** (공짜 탈출 버튼 없음)

**습관·절제 루프**
- **오늘의 할 일 게이트**: 할 일이 남으면 게임 차단 = 게임을 "할 일 다 한 뒤의 보상"으로 (프리맥 원리). 기본 할 일 매일 자동 등록
- **여의주(연속 지킴)**: 약속 지킨 날이 연속 7/14/28일이면 여의주 1/2/3개 → 몽뿔이 → 몽실이 → 몽무기로 진화. **하루라도 한도 초과 시 즉시 리셋 + 퇴화**
- **명상**: 캐릭터를 꾹 누르고 있는 동안만 호흡이 진행(딴짓 방지), 끝까지 채워야 통과
- **심야 가중**: 심야엔 판수·연패 한도가 1씩 낮아짐 (낮 3연패 ≠ 새벽 3연패)
- **명언/다짐**: 내가 적은 문장을 캐릭터가 읊고, 차단 순간에도 띄움. 트래커 종료 시엔 이 문장을 그대로 타이핑해야 꺼짐(최후 탈출구에 마찰)

**리포트·AI**
- **주간 리포트**: 최근 7일 판수/시간/승률/심야 비중/연패 세션/역전패 분석 + 인사이트 한 줄. 지난주 대비 비교, 주차별 영구 저장
- **역전패(near-miss) 감지**: 크게 앞서다 뒤집힌 판 직후(가장 취약한 순간) 캐릭터가 경고
- **AI 판 피드백(선택)**: DeepSeek 키를 넣으면 게임 종료마다 그 판+세션 맥락으로 한마디 (게임당 1콜, 실패 시 조용히 폴백)

**상주·배포**
- 롤 클라가 켜졌을 때만 캐릭터 등장, 평소엔 트레이 상주 (OP.GG Desktop 방식)
- 설정에서 "윈도우 시작 시 자동 실행" → 부팅 시 트레이 상주 ("안 켜면 그만" 구멍 차단)

## 안전 원칙 (핸드오프 문서)
- 게임 파일엔 손 안 댐 — LCU **읽기**만, 내 **자체 창**을 위에 얹기만 (Vanguard 안전, OP.GG가 쓰는 검증된 방식)
- 인게임·챔피언 선택 중엔 절대 개입 안 함 (탈주/닷지 패널티 방지). 로비/매칭 phase에서만
- "완벽한 강제"는 불가능(열쇠는 본인) — 죽이는 게 아니라 **귀찮고 민망하게**. 대사는 매칭 조작 성토가 아니라 도박 심리(본전 추격) 지목

## 실행 (개발)

```
npm install
npm start
```

개발 모드는 `config.json`의 `mode`를 따른다. `mock`이면 조종판 창이 같이 떠서, 롤 없이 phase·통계·여의주를 흉내 내며 모든 상태를 테스트할 수 있다 (조종판 데이터는 파일에 저장 안 됨).

## 빌드 (.exe)

```
npm run pack
```

`dist/LoL Tracker-win32-x64/LoL Tracker.exe` 생성. **패키징된 exe는 항상 real 모드** (조종판 없음, 실제 롤 클라에 붙음). 무설치 폴더라 폴더째 두고 실행하면 되고, 설정에서 자동 실행을 켜면 부팅 시 상주한다.
(정식 NSIS 설치본은 winCodeSign 심볼릭 링크가 Windows 관리자 권한을 요구해 실패 → electron-packager 무설치 빌드를 사용.)

## API 키

- **개발 키**: developer.riotgames.com → 로그인 → 첫 화면 Development API Key 복사 → 설정에 붙여넣기. **24시간마다 만료**(재발급 후 다시 입력).
- **Personal 키(권장)**: "Register Product" → Personal → 만료 없음. 개인 사용엔 이게 정답.
- 키는 `settings.json`에만 저장되고 git에 안 들어감(.gitignore). DeepSeek 키도 동일.

## 구조

```
main.js            메인 프로세스. 창/트레이, 위반 판단(verdict), phase 중계,
                   버튼 막기(클라 창 추적), 여의주·할일·명상·종료 로직
preload.js         렌더러 ↔ 메인 다리
config.json        mode(mock/real), leaguePath
settings.json      사용자 설정 (라이엇/DeepSeek 키, 한도, 할일, 명언 등) — git 제외
state.json         연속 기록·할 일 진행도 — git 제외
reports/           주차별 주간 리포트 아카이브 — git 제외
lcu/real.js        진짜 LCU 연결 (lockfile → WebSocket, client 유무 이벤트)
lcu/mock.js        가짜 클라이언트 (real과 동일한 이벤트 인터페이스)
stats/riot.js      통계 1층 (puuid, 오늘 판수/연패, 타임라인 역전패, 7일 매치)
stats/report.js    주간 리포트 분석 + 인사이트
ai/coach.js        DeepSeek 판 피드백 (폴백 안전)
ui/character.html  캐릭터 창 (표정/애니메이션/막기/명상/드래그)
ui/settings.html   설정 (유리창)
ui/tasks.html      오늘 할 일 (유리창)
ui/report.html     주간 리포트 (유리창)
ui/quit.html       종료 확인 (명언 타이핑)
ui/control.html    mock 조종판 (phase/통계/여의주/연출 버튼)
```

real과 mock은 같은 이벤트(`phase`/`status`/`client`)를 내보내는 쌍둥이라, 나머지 코드는 어느 쪽이 꽂혀 있는지 모른다.

## 집에서 확인할 것 (real 실기)
- 롤 로비에서 한도 초과/할 일 남김 상태로 큐 시도 → 캐릭터가 게임 찾기 버튼을 정확히 덮는지 (`[gate]` 로그로 좌표 확인, 어긋나면 `moveCharacterToButton`의 by 오프셋 조정)
- 클라를 드래그로 옮겨도 방패가 따라붙는지, "할 일 하러 가기"로 클라가 닫히는지
- 명상 완주 / 할 일 완료 시 버튼이 열리는지
- 자동 실행이 재부팅 후 트레이에 뜨는지, DeepSeek 키 넣고 게임 피드백이 오는지
